hexo를 설치하는 방법을 알려주세요

ChatGPT:

  • Hexo는 빠르고 간단한 정적 웹사이트 생성기로, Node.js를 기반으로 합니다. 아래는 Hexo를 설치하는 간단한 방법입니다:

참고: Hexo를 설치하기 전에, 컴퓨터에 Node.js가 설치되어 있어야 합니다. Node.js를 설치하지 않았다면, Node.js 공식 웹사이트(https://nodejs.org)에서 최신 버전을 다운로드하여 설치해 주세요.

  1. npm 설치 확인:
    터미널 또는 명령 프롬프트를 열고, npm (Node.js 패키지 관리자)가 설치되어 있는지 확인합니다. 다음 명령어를 실행하여 npm이 설치되었는지 확인합니다.

npm -v
  1. Hexo 설치:
    터미널 또는 명령 프롬프트에서 다음 명령어를 실행하여 Hexo를 설치합니다.

npm install -g hexo-cli

위 명령을 실행하면, Hexo CLI(Command Line Interface)가 글로벌하게 설치됩니다.

  1. Hexo 프로젝트 생성:
    Hexo CLI가 설치되면, 원하는 디렉토리에서 Hexo 프로젝트를 생성할 수 있습니다. 예를 들어, 웹사이트를 생성할 새로운 디렉토리를 만들고 그 안에서 다음 명령어를 실행합니다.

hexo init my_website
cd my_website
npm install

위의 명령어는 "my_website"라는 이름의 새로운 Hexo 프로젝트를 생성합니다. 프로젝트 디렉토리로 이동하고, 필요한 종속성 패키지를 설치합니다.

  1. Hexo 서버 실행:
    Hexo 프로젝트를 성공적으로 생성한 후, 다음 명령어를 실행하여 개발 서버를 시작합니다.

hexo server

서버를 시작한 후, 웹 브라우저에서 http://localhost:4000로 접속하면 Hexo 웹사이트를 확인할 수 있습니다. 변경 사항을 실시간으로 볼 수 있고, 정적 파일이 생성되는 것을 확인할 수 있습니다.

이제 Hexo를 설치하고 개발 서버를 실행하는 방법을 알게 되었습니다. Hexo는 다양한 테마와 플러그인을 지원하므로, 원하는 방식으로 웹사이트를 커스터마이징할 수 있습니다. 추가적인 정보는 Hexo 공식 문서를 참조하시기 바랍니다.

  1. _config.yml 파일:
    이 파일은 Hexo 웹사이트의 전반적인 설정을 담고 있습니다. 웹사이트의 제목, 설명, 작성자 정보, 언어, 테마 등의 정보를 수정할 수 있습니다.

다음은 일반적으로 수정하게 되는 몇 가지 항목들입니다:

  • title: 웹사이트 제목
  • subtitle: 웹사이트 부제목 또는 설명
  • author: 작성자 이름
  • language: 웹사이트 언어 설정
  • timezone: 웹사이트 시간대 설정
  • theme: 사용할 테마 설정 (테마를 변경하려면 미리 해당 테마를 설치해야 합니다)

수정 방법:
_config.yml 파일을 텍스트 편집기로 열고 원하는 항목들을 수정합니다. 수정 후에는 저장하고 닫습니다.

  1. source/_data/ 디렉토리의 site.js 파일:
    이 파일은 웹사이트의 정보를 더 상세하게 정의하는 파일입니다. _config.yml 파일에서 설정한 몇 가지 정보들은 site.js 파일에서 더 자세하게 표현할 수 있습니다.

다음은 site.js 파일에서 수정할 수 있는 몇 가지 항목들입니다:

  • description: 웹사이트에 대한 더 상세한 설명
  • url: 웹사이트의 URL (주의: 이 설정을 바꿀 때는 기존에 생성된 정적 파일들을 모두 삭제하고 hexo clean 명령을 실행해야 합니다)
  • author: 작성자 정보 (예: 이름, 이메일, 소셜 미디어 등)
  • 기타 등등...

수정 방법:
source/_data/ 디렉토리 안에 있는 site.js 파일을 텍스트 편집기로 열고 원하는 항목들을 수정합니다. 수정 후에는 저장하고 닫습니다.

위의 설정 파일들을 수정한 후에는, 변경 사항이 적용되도록 Hexo를 다시 빌드해야 합니다. 터미널 또는 명령 프롬프트에서 프로젝트 디렉토리로 이동한 다음 다음 명령어를 실행합니다:

  1. 테마 설치:

npm install <테마 npm 패키지 이름> --save

이제 테마를 성공적으로 변경했습니다. 원하는 테마를 찾아서 설치하고 적용하여 멋진 Hexo 웹사이트를 만들어보세요!

---
title: About Me

안녕하세요! 저는 Hexo를 이용하여 만든 웹사이트에 대해 소개하는 페이지입니다. 여기에 자기소개를 작성하세요!

hexo clean  # 기존 정적 파일들을 삭제

이제 About 페이지를 성공적으로 만들었습니다. 원하는 방식으로 About 페이지를 디자인하고 내용을 추가하여 웹사이트에 더 많은 정보를 제공할 수 있습니다.

ChatGPT:

  • Hexo에서 hexo generate 명령을 실행하면, 정적 웹사이트 파일이 생성되는 폴더는 기본적으로 Hexo 프로젝트의 루트 디렉토리 안에 있는 public 폴더입니다. 이 폴더에는 HTML, CSS, JavaScript 및 이미지와 같은 모든 웹사이트 파일이 포함됩니다.

정적 웹사이트 파일이 생성된 후에는 웹서버를 실행하거나 해당 폴더의 내용을 웹 호스팅 서비스 또는 웹 서버에 업로드하여 웹사이트를 온라인으로 공유할 수 있습니다.

기본적으로 hexo generate 명령을 실행하면 Hexo는 프로젝트의 source 디렉토리 안에 있는 Markdown 파일들과 테마의 레이아웃, CSS, JavaScript 등을 사용하여 웹사이트를 빌드하고 public 폴더에 결과물을 저장합니다.

이후 hexo server 명령을 실행하면, Hexo는 public 폴더의 내용을 웹 서버로 제공하여 로컬에서 웹사이트를 확인할 수 있습니다. hexo server를 실행한 후에는 브라우저에서 http://localhost:4000으로 접속하여 웹사이트를 확인할 수 있습니다.

따라서 정적 웹사이트 파일은 기본적으로 public 폴더 안에 생성되며, 이 폴더의 내용을 웹서버나 웹 호스팅 서비스에 업로드하여 웹사이트를 호스팅할 수 있습니다.

다음은 A라는 메뉴를 선택하면 A라는 메뉴에 속하는 포스트만 가져오는 방법의 예시입니다:

  1. Front Matter에 메뉴 정보 추가:
    A 메뉴에 속하는 포스트들에게 Front Matter를 이용하여 메뉴 정보를 추가합니다. 각 포스트의 Markdown 파일에서 Front Matter 영역에 menu 항목을 추가합니다.

예를 들어, A 메뉴에 속하는 포스트에 menu: A라는 정보를 추가하려면 아래와 같이 작성합니다:


---
title: A 포스트
date: 2023-08-06
menu: A
---

위와 같이 menu 항목에 메뉴 이름을 지정해주면 해당 포스트가 A 메뉴에 속하게 됩니다.

  1. A 메뉴에서 포스트 필터링:
    A 메뉴에서 A에 해당하는 포스트들만 가져오려면 해당 메뉴를 표시하는 템플릿 파일에서 필터링 기능을 사용하여 포스트를 선택합니다. 대부분의 테마에서는 메뉴에 해당하는 템플릿 파일이 _config.yml 파일에 지정된 대로 사용됩니다.

예를 들어, A 메뉴를 표시하는 menu_A.ejs라는 템플릿 파일이 있다면, 해당 파일에서 A 메뉴에 속하는 포스트를 가져오기 위해 다음과 같이 필터링 기능을 사용합니다:


<% var postsInMenuA = page.posts.filter(post => post.menu === 'A'); %>

<% postsInMenuA.forEach(post => { %>
  <!-- 포스트를 템플릿에 표시하는 코드 -->
<% }); %>

위 코드에서는 page.posts.filter(post => post.menu === 'A')로 A 메뉴에 속하는 포스트들만 필터링하여 postsInMenuA라는 변수에 저장합니다. 그리고 postsInMenuA.forEach(...)로 필터링된 포스트들을 순회하며 템플릿에 표시하는 코드를 작성합니다.

이렇게 필터링 기능을 사용하여 A 메뉴에 속하는 포스트들만 템플릿에 표시하면, A 메뉴를 선택하면 해당 메뉴에 속하는 포스트만 가져올 수 있습니다. 필요에 따라 테마의 템플릿 파일을 커스터마이징하여 원하는 메뉴별로 포스트를 필터링할 수 있습니다.
